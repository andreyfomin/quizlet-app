# AI Quizlet — Slackbot

A Slack app (Java, Spring Boot 3 + [Slack Bolt for Java](https://docs.slack.dev/tools/java-slack-sdk/)) that
lets a Slack workspace create and play the quizzes generated by the [backend](../backend).
See [`ARCHITECTURE.md`](ARCHITECTURE.md) for the design rationale, package layout, and
sequence diagrams — this file is the step-by-step "how do I actually run this" guide.

## What it does

| Slack command                                    | What happens                                                              |
|----------------------------------------------------|------------------------------------------------------------------------------|
| `/quiz create <topic>`                              | Calls the backend to generate a quiz, replies with its id                    |
| `/quiz create <topic> \| <questionCount>`           | Same, with a custom question count instead of the backend's default (5)      |
| `/quiz list`                                         | Lists every quizlet that exists (id, topic, question count)                  |
| `/quiz delete <quizletId>`                           | Deletes a quizlet — fails if any session has ever used it                    |
| `/quiz start <quizletId> @user1 @user2 ...`         | Starts a session for those users; DMs each of them question 1 with buttons   |
| *(click an answer button)*                          | Submits the answer to the backend, updates the DM with feedback + next question — on the last question, shows final score (X/Y correct) and a full right/wrong breakdown instead |
| `/quiz progress <sessionId>`                        | Posts each participant's score / completion in the channel                   |
| `/quiz help` (or a blank/unrecognized command)       | Lists all available commands                                                 |

It's delivered over Slack's **HTTP Events API** (not Socket Mode): Slack POSTs
slash commands and button clicks to one endpoint, `POST /slack/events`, which this
app exposes on port `8081`.

## Prerequisites

- A Slack workspace where you can install apps (or create a free one at
  [slack.com/create](https://slack.com/create) for testing).
- The [backend](../backend) module running somewhere reachable from this app
  (`BACKEND_BASE_URL`, default `http://localhost:8080`).
- For local development: [ngrok](https://ngrok.com/download) (or any other way to
  expose `localhost:8081` over HTTPS) — Slack requires a public HTTPS URL to deliver
  events to, it will not call `http://localhost` directly.

## 1. Start ngrok (local dev only)

Slack needs a URL to reach you before you can finish creating the app, so start this
first and keep it running:

```sh
cd ..
./gradlew ngrokStart
```

(from the repo root — plain `ngrok http 8081` works the same, the task just also
detects and reuses an already-running tunnel instead of starting a duplicate).

Copy the `https://<random>.ngrok-free.app` URL it prints — you'll paste it into the
manifest below as `<YOUR_PUBLIC_URL>`. If you redeploy the app to a real domain later
(docker-compose behind a real reverse proxy, etc.), that domain replaces this ngrok
URL in the two places it's used (Slash Commands request URL, Interactivity request
URL) — see "Going to production" at the bottom.

Free ngrok assigns a **new random URL every restart**, which normally means
re-editing the Slack app config each time you come back to develop. Step 3 below
sets up automatic sync for that — see "Keep the Slack app URL in sync
automatically" further down.

## 2. Create the Slack app

The fastest way is from an **app manifest**, which sets scopes, the slash command,
and interactivity in one step instead of clicking through several dashboard screens.

1. Go to <https://api.slack.com/apps> → **Create New App** → **From an app manifest**.
2. Pick the workspace you want to install it into.
3. Paste this manifest, replacing every `<YOUR_PUBLIC_URL>` with the ngrok URL from
   step 1 (e.g. `https://a1b2c3d4.ngrok-free.app`):

   ```yaml
   display_information:
     name: AI Quizlet
     description: Create and play AI-generated quizzes with your team
   oauth_config:
     scopes:
       bot:
         - commands
         - chat:write
         - im:write
   settings:
     interactivity:
       is_enabled: true
       request_url: <YOUR_PUBLIC_URL>/slack/events
     org_deploy_enabled: false
     socket_mode_enabled: false
     token_rotation_enabled: false
   features:
     bot_user:
       display_name: AI Quizlet
       always_online: true
     slash_commands:
       - command: /quiz
         url: <YOUR_PUBLIC_URL>/slack/events
         description: Create and play AI-generated quizzes
         usage_hint: "create <topic> | list | delete <id> | start <id> @user... | progress <sessionId> | help"
         should_escape: true
   ```

4. Click through the preview screen — Slack validates the manifest live and will
   flag any field it doesn't recognize, so if something's off here it'll tell you
   exactly what.
5. **Create**.

   *(If you'd rather not use a manifest: create a blank app, then under **OAuth &
   Permissions** add the three bot scopes listed above, under **Slash Commands**
   add `/quiz` pointing at `<YOUR_PUBLIC_URL>/slack/events`, and under
   **Interactivity & Shortcuts** turn it on with the same request URL. Same end
   result.)*

## 3. Install the app and collect credentials

1. **OAuth & Permissions** → **Install to Workspace** → **Allow**.
2. Copy the **Bot User OAuth Token** (starts `xoxb-`) from that same page.
3. **Basic Information** → **App Credentials** → copy the **Signing Secret**.

**Optional, one-time, while you're here** — enables the automatic URL sync
described below instead of manually re-editing the app config on every ngrok
restart:

4. Same **Basic Information** page → copy the **App ID**.
5. Go to <https://api.slack.com/apps> (your apps list, not this app's pages) →
   scroll to **Your App Configuration Tokens** → **Generate Token** → copy the
   **Refresh Token** (starts `xoxe-`).

## 4. Configure environment variables

Put these in the repo root `.env` (this file is git-ignored — see
[`../.env.example`](../.env.example) for the full list of variables):

```dotenv
SLACK_BOT_TOKEN=xoxb-...
SLACK_SIGNING_SECRET=...

# optional — only if you did step 3's config-token setup:
SLACK_APP_ID=A0...
SLACK_CONFIG_REFRESH_TOKEN=xoxe-...
```

`App()` (Bolt's entry point, wired in
[`SlackAppConfig`](src/main/java/com/aiquizlet/slackbot/config/SlackAppConfig.java))
reads `SLACK_BOT_TOKEN` / `SLACK_SIGNING_SECRET` automatically — nothing else to
configure for the bot itself. The other two are read by the root
`slackManifestSync` Gradle task, not the app.

## 5. Run it

**Everything together (recommended for trying it end to end):**

```sh
cd ..
./gradlew composeUp   # or: docker compose up --build -d
./gradlew composePs   # or: docker compose ps
```

This starts Postgres, the backend, and the slackbot together, wired via the `.env`
file (see the repo root [`docker-compose.yml`](../docker-compose.yml)). The slackbot
container reaches the backend at `http://backend:8080` automatically; ngrok still
needs to point at your host's port `8081`. See the root
[`README.md`](../README.md#5-build-and-start-everything) for the full
walkthrough of this path end to end.

**Slackbot only, against a backend you're already running:**

```sh
cd slackbot
export $(grep -v '^#' ../.env | xargs)   # loads SLACK_BOT_TOKEN, SLACK_SIGNING_SECRET, etc. into the shell
../gradlew bootRun
```

Or build and run the jar directly (matches how the backend is run in
[`../backend/ARCHITECTURE.md`](../backend/ARCHITECTURE.md)):

```sh
../gradlew :slackbot:bootJar
java -jar build/libs/slackbot-0.0.1-SNAPSHOT.jar
```

Either way, confirm it started by checking the servlet responds (it should reject
an unsigned request with `400 Invalid Request` — that means it's alive and
verifying signatures correctly, not misconfigured):

```sh
curl -i -X POST http://localhost:8081/slack/events -d "probe=1"
```

## 6. Try it in Slack

```
/quiz create Java Streams
```
→ "Generating your quiz..." immediately, then a follow-up "Created quizlet #1 on
*Java Streams* (5 questions). Start it with `/quiz start 1 @user1 @user2 ...`"
within a few seconds.

```
/quiz start 1 @alice @bob
```
→ starts a session and DMs both alice and bob question 1 with four answer buttons.

Click a button in the DM → the message updates in place with ✅/❌ feedback and the
next question, until all questions are answered.

```
/quiz progress 1
```
→ posts each participant's score and completion status to the channel.

## Keep the Slack app URL in sync automatically

If you did the optional part of step 3, restarting ngrok is one command
instead of manually re-editing the Slack app's Request URLs:

```sh
cd ..
./gradlew slackManifestSync
```

This starts ngrok if it isn't already running (same as `ngrokStart`), then
pushes the current tunnel URL into the Slack app via `apps.manifest.update`,
using [`slack-manifest.json`](slack-manifest.json) (the JSON twin of the YAML
manifest from step 2 — edit that file, not the YAML, if you change scopes or
the slash command going forward, since this is what actually gets pushed).

The config access token this needs expires every 12 hours, so the task
rotates it on every run and writes the new refresh token back into `.env`
automatically — don't restore an old backup of `.env` over a newer one, or
you'll invalidate the token that's actually current. If you ever see
`invalid_refresh_token`, generate a fresh pair (step 3.5) and update `.env`.

This can't create the app itself — only push updates to one that already
exists (step 2 is still a one-time manual step).

## Environment variables

| Variable                | Required | Default                   | Purpose                                          |
|---------------------------|:--------:|------------------------------|----------------------------------------------------|
| `SLACK_BOT_TOKEN`         | ✅       | —                             | Bot token from step 3, used for all Slack Web API calls (DMs, updates) |
| `SLACK_SIGNING_SECRET`    | ✅       | —                             | Verifies incoming requests really came from Slack |
| `BACKEND_BASE_URL`        |          | `http://localhost:8080`      | Where the backend REST API lives                  |
| `SERVER_PORT`             |          | `8081`                       | Port this app listens on (Spring's standard `server.port`, override via `SERVER_PORT` env or `-Dserver.port=`) |
| `SLACK_APP_ID`            |          | —                             | Used by `./gradlew slackManifestSync` (not read by the app itself) — see "Keep the Slack app URL in sync automatically" |
| `SLACK_CONFIG_REFRESH_TOKEN` |       | —                             | Same — also **rewritten automatically** by that task after every run |

## Troubleshooting

| Symptom                                                        | Likely cause                                                                 |
|-------------------------------------------------------------------|----------------------------------------------------------------------------------|
| `/quiz failed because the app did not respond` / "This app took too long to respond" | Slack got no HTTP response at all — almost always the Request URL Slack has on file doesn't point at your current, running ngrok tunnel. **Most common cause: ngrok was restarted and got a new URL.** Run `./gradlew slackManifestSync` to push the current one (see "Keep the Slack app URL in sync automatically"), or manually re-check **Slash Commands** / **Interactivity & Shortcuts** against `curl -s http://127.0.0.1:4040/api/tunnels`. Only if the URL is confirmed correct and this still happens: the handler didn't `ack()` within 3s (`create`/`start` already ack immediately and finish in the background, so this would point at the app itself, not the URL). |
| `invalid_refresh_token` from `slackManifestSync`                | The `SLACK_CONFIG_REFRESH_TOKEN` in `.env` is stale or was already used (each one is single-use, rotated on every run) — regenerate a pair at **Your App Configuration Tokens** (step 3.5) and update `.env`. |
| `400 Invalid Request` shown to the *user* in Slack (not curl)   | Signing secret mismatch — the `SLACK_SIGNING_SECRET` env var doesn't match what's on the **Basic Information** page. Re-copy it. |
| Slash command works but no DM arrives                           | Missing `im:write` scope, or the app hasn't been reinstalled after adding a scope — **OAuth & Permissions** → **Reinstall to Workspace**. |
| `/quiz create` replies with `:x: Failed to create quiz: ...`    | The backend call failed — check the backend is reachable at `BACKEND_BASE_URL` and (if the error mentions a 502) that the backend's own `HF_API_TOKEN` is set. See [`../backend/ARCHITECTURE.md`](../backend/ARCHITECTURE.md). |
| Works via `curl` to `/slack/events` but not from real Slack     | You're hitting the servlet directly, bypassing signature verification — that only confirms the servlet is *up*, not that Slack can reach it. Re-check the ngrok URL matches what's configured in the Slack app (or run `slackManifestSync`). |

## Going to production

Everything above assumes local development with ngrok. For a real deployment:

- Point the Slash Commands / Interactivity request URLs at your real HTTPS domain
  instead of the ngrok URL (**Slash Commands** and **Interactivity & Shortcuts**
  pages in the Slack app config).
- ngrok's URL changes every time you restart it (free tier) — during
  development, `./gradlew slackManifestSync` handles re-pointing the Slack
  app at the new URL (see above); this doesn't apply once you're on a real
  domain.
- Consider Socket Mode instead of HTTP if the bot should run entirely inside a
  private network with no public inbound endpoint at all — that's a different
  transport (`bolt-socket-mode` instead of `bolt-jakarta-servlet`) and wasn't the
  mode chosen here, but the command/action handler code
  (`QuizCommandService`, `AnswerActionService`) would carry over unchanged.
